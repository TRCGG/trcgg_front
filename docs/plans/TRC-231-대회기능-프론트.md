# TRC-231 대회 기능 — 프론트 (대회 관리 화면 + 전적/순위 UI)

> 티켓: [TRC-231](https://trcgg.atlassian.net/browse/TRC-231) (프론트 파트)
> 공통 우산: [TRC-229](https://trcgg.atlassian.net/browse/TRC-229) `[공통] 대회 기능 도입`
> 백엔드 파트: TRC-230 (`loltrixbe/loltrix_be/docs/plans/TRC-230-대회기능-백엔드.md`) — API 계약은 이 문서 기준

## 배경 / 문제

일반내전 전적만 보여주는 현 서비스에 **대회** 화면을 추가한다.
AI 인터뷰(2026-07-08)로 확정된 명세 중 프론트 관련:

- guild_manager가 **웹에서** 대회 개설·팀/로스터 등록·종료를 수행 (인터뷰 결정: 관리는 웹)
- **웹에서도 코드 발급** 가능 (발급 시 대회·스크림/본경기 구분 선택, 코드를 화면에 표시)
- 대회별로 스크림/본경기 **분리 전적**과 **승수 집계 순위표** 조회 — 순위표에는
  "공식 순위 아님(단순 집계)" 명시 (인터뷰 결정: 형식 엔진 없음)
- 로스터 불일치 경기는 미분류로 남음 → 운영자가 수동 보정하는 UI 필요

인터뷰 후속 확정 (2026-07-08 기획 검토):

- 라이프사이클 **3단계 준비→진행→종료**: 준비 중엔 팀/로스터 등록만, 발급은 진행 상태에서만
- 순위표는 **스크림/본경기 각각** 제공
- 웹 발급 코드는 **화면 표시만** (봇 채널 게시 없음 — 확정)
- **종료된 대회에서도 미분류 보정은 가능** (발급·팀 수정만 차단)

기존 참고 화면: TRC-223 멤버 권한 관리 화면(guild_manager 전용 화면 패턴) —
권한 분기·가드 패턴을 재사용한다.

피처 브랜치에만 있어(확정 기능 아님, 테스트 단계) dev에 머지된 뒤 착수한다.

## 목표 / 비목표

**목표**

1. 대회 관리 화면 (guild_manager 전용): 개설·종료, 팀/로스터 CRUD, 웹 코드 발급, 미분류 보정
2. 대회 조회 화면 (공개): 대회 목록/상세 — 구분별 전적 + 순위표
3. 권한 분기: 비 guild_manager에게 관리 UI 비노출 + API 에러 처리

**비목표**

- 대진표/일정 UI, 밴픽·챔피언 통계 UI (다음 버전)
- 기존 일반 전적 화면 변경 (대회는 독립 메뉴)

## 변경 범위 (화면·컴포넌트) — 잠정, 단계 1 실사 후 확정

| 화면 | 내용 |
|---|---|
| 대회 목록 | 준비/진행/종료 상태 표시, 상세 진입 |
| 대회 상세 | 탭: 본경기 전적 / 스크림 전적 / 순위표(본경기·스크림 각각) / (관리자) 관리 |
| 순위표 | 구분별 승수·승률 정렬 테이블 + "단순 집계" 배지 |
| 관리 탭 | 상태 전이(준비→진행→종료), 팀/로스터 편집(종료 전), 코드 발급(진행 상태에서만, 구분 선택→코드 표시·복사), 미분류 경기 보정(종료 후에도 가능) |

- API 연동: 공통 request 모듈 재사용 (TRC-50 패턴)
- 상세 라우팅·컴포넌트 구조는 단계 1에서 기존 페이지 구조 실사 후 확정

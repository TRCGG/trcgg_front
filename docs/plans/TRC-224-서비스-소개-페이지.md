## 배경 / 문제

Riot 프로덕션 키 심사 요건: "검증 가능한 서비스 소개 공개 웹사이트 + 유저 플로우 설명".
현재 홈(`/`)은 검색 도구형이라 심사관이 서비스 목적을 파악하기 어렵다.

repo 실사(2026-07-07): Next.js 12 Pages Router, 인증 가드 전무(모든 페이지 공개),
Tailwind 3, 공용 Layout 래퍼 없음(페이지가 자체 조립), i18n 없음, 이미지는
`src/assets/images/` + `next/image` import 관례.

## 목표 / 비목표

**목표**: `/about` 공개 소개 페이지 1장 — 심사관이 읽고 서비스를 검증할 수 있는 수준

**비목표**: 홈(`/`) 변경, i18n 프레임워크 도입(영문은 정적 병기), 디자인 시스템 개편

## 변경 범위

| 파일 | 내용 |
|---|---|
| `src/pages/about.tsx` (신규) | 소개 페이지. Tailwind 유틸리티, 기존 색상(`src/styles/colors.js`)·폰트 준수 |
| `src/components/layout/NavBar.tsx` | `navList`에 `/about` 링크 추가 (최소 변경) |
| `src/assets/images/about/` (신규 폴더) | 유저 플로우·기능 스크린샷 (스크린샷은 후속 제공 — 슬롯 먼저) |

## 페이지 구성 (§3 명세 준수, 각 섹션 한/영 병기)

1. **Hero** — 서비스 한 줄 소개: "길드 내전 전적·통계 서비스 (무료)" + 기존 mainLogo 재사용
2. **유저 플로우** — 5단계 카드: ① 디스코드 내전 시작 → ② 봇이 토너먼트 코드 게시 →
   ③ 클라이언트에서 코드 입장 → ④ 경기 후 자동 기록 → ⑤ 웹에서 전적 확인.
   현행 리플 업로드 플로우 병기. 각 단계에 스크린샷 슬롯(제공 전엔 아이콘/플레이스홀더).
3. **주요 기능** — 전적/승률/랭킹, MMR, 상대전적(H2H) 카드
4. **운영 정보** — 무료·비상업 명시, 문의(디스코드), Riot 고지문:
   "gmok isn't endorsed by Riot Games and doesn't reflect the views or opinions of
   Riot Games or anyone officially involved in producing or managing League of Legends."
5. **SEO/메타** — 페이지 자체 `<Head>`로 title/description (한/영)

## 영향받는 불변식 / 리스크

- 홈·기존 페이지 무수정 (NavBar 링크 1줄 제외)
- 인증 가드가 없으므로 별도 공개 처리 불필요 — 단, 소개 페이지에 로그인 요구 요소를 넣지 않는다
- 스크린샷 미제공 상태로도 페이지가 성립해야 함 (심사 제출 전 실스크린샷 필수)

## 검증 방법

- `npm run build` 통과, `npm run dev`에서 `/about` 비로그인 렌더 확인
- 모바일 뷰포트(Tailwind responsive) 확인
